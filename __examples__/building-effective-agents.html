Building Effective Agents

A practical guide to building reliable AI agents based on Anthropic's research and experience working with dozens of teams

01 / Introduction

Simplicity Over Complexity

Over the past year, we've worked with dozens of teams building large language model (LLM) agents. We consistently found that the most successful implementations weren't using complex frameworks or specialized libraries. Instead, they were building with simple, composable patterns.

This guide shares the best practices we've developed while helping teams deploy agents in production, along with practical insights from our own agent development.

Successful agentic systems use simple, composable patterns rather than complex frameworks.

02 / Core Concepts

Workflows vs. Agents

What are Workflows?

Workflows are systems where LLMs and tools are orchestrated through predefined code paths. In this pattern, developers explicitly specify the steps and order of execution, with LLMs handling specific subtasks at each step.

What are Agents?

Agents are systems where LLMs dynamically direct their own processes and tool usage. They maintain control over how they accomplish tasks, deciding when to call tools, how to interpret results, and when to conclude the task.

"Workflows are suitable for predictable, structured tasks; Agents excel at open-ended problems requiring flexible decision-making."
03 / Decision Guide

When to Use Workflows vs. Agents

Use Workflows

Predictable tasks with clear steps, strict control requirements, cost-sensitive scenarios

Use Agents

Flexible decision-making needed, uncertain steps, open-ended problems, autonomous exploration required

Hybrid Approach

Many successful applications combine both: workflows for structured parts, agents for decision points

Key Indicators

If failure is costly or audit trails are required, prefer workflows

04 / Building Blocks

Core Components of Agents

1. Planning

Agents need to break down complex tasks into manageable steps. This can be achieved through chain-of-thought prompting or dedicated planning modules.

2. Memory

Effective agents need to maintain context across interactions. This includes conversation history, intermediate results, and key facts. Memory can be short-term (within session) or long-term (across sessions).

3. Tool Use

Tools extend LLM capabilities by enabling interaction with external systems. The key is providing agents with appropriate tools and clearly describing their purpose.

Best Practice: Provide clear descriptions, usage examples, and error handling for tools. The quality of tool descriptions directly impacts agent performance.

05 / Design Patterns

Effective Agentic Patterns

Prompt Chaining

Decompose tasks into a sequence of prompts, each processing the previous output. Simple and easy to debug.

Routing

Classify inputs and route to specialized sub-flows. Ideal for handling diverse input types.

Parallelization

Run multiple LLM calls simultaneously and aggregate results. Good for tasks requiring multi-angle analysis.

Evaluator-Optimizer

One LLM generates output while another evaluates and provides feedback in a loop. Ideal for high-quality requirements.

Agent Pattern

LLM decides when to call tools and interpret results autonomously. Suitable for open-ended problems.

Multi-Agent

Multiple agents collaborate on tasks, each handling different aspects. Good for complex, multi-dimensional problems.

06 / Best Practices

Recommendations for Building Reliable Agents

1. Start Simple

Don't build complex agents from the start. Begin with the simplest solution and add complexity only when necessary. Often, a well-designed prompt chain is sufficient.

2. Design Good Tool Interfaces

Tool descriptions should be as clear as documentation for a junior developer. Include usage examples, edge cases, and error handling. Tool interface quality directly impacts agent reliability.

3. Provide Good Feedback Mechanisms

When agents make mistakes, they should receive feedback and correct. Design systems with error recovery and retry mechanisms in mind.

"Invest in tool design and error handling. This improves agent reliability more than prompt optimization alone."
07 / Appendix

Prompt Engineering Your Tools

Tools are a crucial part of any agentic system. They enable Claude to interact with external services through well-defined interfaces. Tools should receive the same prompt engineering attention as your overall prompts.

Key Principles

Give models room to "think" before they write themselves into a corner. Some formats are much harder for LLMs than others—writing diffs requires knowing line counts, while JSON requires escaping.

Golden Rule: Invest as much effort in creating good agent-computer interfaces (ACI) as you would in human-computer interfaces (HCI).

How to Design Good Tools

• Put yourself in the model's shoes—is tool usage obvious from the description?
• Change parameter names to make things more obvious
• Test extensively in the workbench to see what mistakes the model makes
• Apply poka-yoke principles: make it harder to make mistakes

08 / Conclusion

Simplicity is Key

Building effective AI agents doesn't require complex frameworks or specialized libraries. The most successful implementations use simple, composable patterns, focusing on good tool design and clear interfaces.

Start simple, add complexity only when necessary. Invest in tool design and error handling—this improves reliability more than prompt optimization.
